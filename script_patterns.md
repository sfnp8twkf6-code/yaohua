# Script Patterns

账号核心句：

> 用普通人能听懂的方式，拆开一个生意为什么成立。

## 主栏目：商业模式出海拆解

适合案例：瑞幸、蜜雪冰城、名创优品、泡泡玛特、SHEIN、Temu、TikTok Shop、海底捞、比亚迪、安克创新。

固定结构：

1. 反常识开头：观众以为 A，但真正值得看的是 B。
2. 国内成立原因：这家公司在中国赢的不是单点产品，而是一套结构。
3. 出海变量：换市场后，哪些条件消失，哪些能力还能迁移。
4. 商业判断：它出海成败取决于哪个关键变量。
5. 普通人启发：这件事说明了什么商业规律。

## 脚本骨架

```text
很多人以为「案例」是在做「表层动作」。
但我觉得，真正值得看的不是「表层动作」，而是「底层结构」。

它在国内赢，靠的不是单点优势，而是三件事：
第一，...
第二，...
第三，...

但一旦出海，问题会变成：
原来那套结构，哪些还能复制？哪些必须重做？

所以这家公司真正的考题不是「能不能把产品卖出去」，
而是「能不能把国内那套效率系统，翻译成海外用户愿意接受的体验」。

这也是中国品牌出海最难的地方：
不是把货搬出去，而是判断你在国内赢的那套结构，换个市场之后还剩多少有效。
```

## 视觉模式

优先使用素材、动效、结构图，不露脸。

推荐画面比例：

- 40% 案例素材：门店、产品、海外社媒截图、官网/财报关键页。
- 40% 结构动画：流程图、变量拆分、箭头、对比表。
- 20% 金句卡片：黑白高对比、关键字放大、短暂停顿。

## 禁区

不要做“商业知识百科”，太像课件。

不要做“外贸大师口吻”，信任度不够。

不要堆数据吓人，数据只服务于判断。

不要每条都讲出海，栏目可以扩展到“一个生意为什么成立”。
