# WRITE-002: 通用人味口播生成

平台：ChatGPT / Claude / Gemini / DeepSeek / 豆包 / Kimi

作用：当你只想用一个平台完成初稿时，用这条提示词。

适合场景：你有选题、有一点想法，希望 AI 帮你一次性产出标题、框架、口播稿和审稿建议。

最后更新：2026-06-27

## 可修改变量

- `【选题】`
- `【我的观点】`
- `【资料】`
- `【目标受众】`
- `【目标时长】`

## 可复制提示词

```text
你现在是我的短视频内容主编，不是普通文案助手。

我的账号方向：
用普通人能听懂的方式，拆解商业案例、品牌出海、财经现象和商业结构。内容要有观点、有故事、有启发，不做空泛鸡汤，也不做资料搬运。

这次选题：
【选题】

我的观点或想法：
【我的观点】

我已有的资料：
【资料】

目标受众：
【目标受众】

目标时长：
【目标时长】

请你按下面流程帮我完成：

第一步，提炼核心判断：
- 用一句话说清楚这条视频真正想表达什么。
- 判断这个选题是否适合短视频传播。
- 如果不够尖锐，请帮我把角度改得更有冲突和吸引力。

第二步，给我 10 个标题：
- 3 个偏商业洞察
- 3 个偏反常识
- 2 个偏电影解说感
- 2 个偏大众好奇心
标题不能标题党，不能虚假夸张，不能用“震惊、炸裂、颠覆认知”这种廉价词。

第三步，选出你最推荐的 3 个标题，并说明为什么。

第四步，给我一版短视频文案框架：
- 开头 5 秒：必须有悬念或反常识。
- 第一段：大众以为是什么。
- 第二段：真正值得看的是什么。
- 第三段：商业结构拆解。
- 第四段：高潮判断。
- 第五段：普通人的生活启发。
- 结尾：一句有记忆点的话。

第五步，写一版 90-120 秒口播稿：
- 像人说话，不要像 AI 写作。
- 不要出现“首先、其次、最后”这种课件感结构。
- 不要频繁使用“本质上、值得注意的是、不仅是更是、换句话说”等 AI 常用句。
- 可以有个人判断，例如“我更关心的是”“我觉得这里真正有意思的是”。
- 要有故事推进感，不能只是观点罗列。
- 每 20 秒左右要有一次信息推进或反转。
- 结尾要让普通人觉得：这件事和我也有关。

第六步，最后帮我做一次审稿：
- 哪句话最像 AI，帮我改掉。
- 哪段最无聊，帮我压缩。
- 哪个观点最有传播价值，帮我放大。
- 哪个地方缺少个人判断，帮我补强。
- 给这条内容打一个 1-10 分的短视频传播潜力分，并说明扣分原因。

请直接输出结果，不要解释你会怎么做。
```

## 二次追问

```text
这版还是太顺、太完整、太像 AI。请你保留核心观点，但把表达改得更像一个真实的人在跟朋友讲一个刚想明白的商业故事。允许有一点停顿感、转念感和个人判断。少一点结论腔，多一点“我为什么这么看”的思考过程。
```
